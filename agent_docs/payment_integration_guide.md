# 个人开发者在线支付接入指南

对于个人开发者而言，接入在线支付主要面临资质（营业执照）的门槛。以下是目前（2025年）主流的可行方案，分为**国内收款**和**国际收款**两类。

## 一、国内收款方案

### 1. 注册个体工商户（最推荐，正规稳定）
这是目前最稳妥、合规的长期方案。
- **原理**：微信支付和支付宝都支持“个体工商户”类型的商户申请。
- **流程**：
    1. 在线或线下办理个体工商户营业执照（现在很多地区支持全程网办，如淘宝店主资质等）。
    2. 使用营业执照在[微信支付商户平台](https://pay.weixin.qq.com)和[支付宝商家中心](https://b.alipay.com)开户。
    3. 获批后即可获取 API 密钥，进行正规开发。
- **优点**：官方接口，资金直接入账，费率标准（通常 0.6% 左右），稳定不封号。
- **缺点**：需要报税（虽然小微企业有免税额度，但仍需按时申报）。

### 2. 第三方聚合支付/免签服务（适合短期/试错）
有一些服务商专门为个人开发者提供支付接口对接，它们通常持有企业资质对接官方，再分发给个人。
- **代表平台**：YunGouOS、PayJs、虎皮椒等。
- **原理**：平台作为服务商，为你开通分商户或代收款。
- **优点**：接入简单，通常有现成的 SDK 和插件，无需营业执照（部分平台）。
- **缺点**：
    - **费率高**：通常在 1% - 3% 甚至更高。
    - **资金风险**：资金可能先经过平台，存在跑路风险（尽量选择资金直连的）。
    - **合规性**：存在违规被微信/支付宝清退的风险。

### 3. 个人收款码 + 监听 App（**极不推荐**）
通过安卓 App 监听手机通知栏的收款消息，回调服务器。
- **缺点**：极其不稳定（掉单），不仅体验差，而且容易被微信/支付宝风控系统识别为异常交易导致**限制收款甚至封号**。**强烈建议不要在生产环境使用。**

## 二、国际收款方案（适合数字产品/SaaS）

如果你的产品面向全球用户，或者主要销售虚拟商品，国际方案可能更灵活。

### 1. Paddle（最推荐）
Paddle 是各类独立开发者的首选。它不仅是支付网关，更是 Reseller（经销商）。
- **优点**：
    - **Merchant of Record (MoR)**：Paddle 帮你处理全球税收（VAT/GST），你不需要担心各国税务问题。
    - **支持中国**：明确支持中国开发者注册。
    - **支持支付宝**：结算页面可以让用户选择 Alipay，即使你没有支付宝商户号。
- **缺点**：费率较高（约 5% + $0.50），审核较严（需要有完成度较高的网站）。

### 2. Lemon Squeezy
类似 Paddle 的 MoR 服务，界面更现代，对开发者友好。
- **优点**：支持订阅、许可证验证等功能，非常适合卖软件或 SaaS。
- **注意**：需通过 PayPal 或 Payoneer 等方式提现，注册时需确认其对中国地区的支持政策（政策常变动）。

### 3. Stripe Atlas / Incorpuk
- **原理**：通过 Stripe Atlas 在美国或 Incorpuk 在英国注册一家公司。
- **优点**：可以申请标准的 Stripe 账号，生态最强。
- **缺点**：成本高（注册费+维护费+报税），适合业务量较大后考虑。

---

## 三、方案对比与建议

| 方案 | 适用场景 | 难度 | 成本 | 稳定性 | 推荐指数 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **个体工商户** | 国内业务，长期运营 | 中 | 低 (办证免费) | 高 | ⭐⭐⭐⭐⭐ |
| **Paddle** | 国际业务，数字产品 | 低 | 中 (费率高) | 高 | ⭐⭐⭐⭐⭐ |
| **聚合支付** | 验证 MVP，短期项目 | 低 | 中 | 中 | ⭐⭐⭐ |
| **注册海外公司** | 业务规模化，拓展全球 | 高 | 高 | 高 | ⭐⭐⭐ |

**总结建议**：
如果你的主要受众在国内，建议花点时间去**注册一个个体工商户**，这是一劳永逸的办法。
如果你的产品是面向全球的（如独立的工具站、插件），直接接入 **Paddle** 是最省心的选择。
